# ZeroSeg: Cross-Platform Security Suite v10.0

Программный комплекс ZeroSeg представляет собой кроссплатформенное решение для автоматизированного аудита информационной безопасности (ИБ) и управления сетевым периметром хоста. Система ориентирована на обеспечение комплаенса в рамках нормативного поля РФ.

---

## 1. Техническая архитектура и UML

Система спроектирована с использованием объектно-ориентированного подхода и паттерна «Абстрактный интерфейс» для обеспечения переносимости между ядрами NT и Linux.

### Архитектура кода (UML Class Structure):
- **ZeroSegScanner:** Базовый класс-контроллер.
  - *Attributes:* `vector<string> vuln_ports`, `vector<string> vuln_sites`.
  - *Methods:* `initConsole()`, `scanPorts()`, `managePorts()`, `showLegalInfo()`.
- **System Wrappers:** Инкапсуляция вызовов `system()` для `netsh` (Windows) и `iptables` (Linux).



### Схема обработки сетевого трафика (Sequence Diagram):
1. Модуль сканирования инициирует TCP-запрос (SYN) к локальным портам.
2. При получении ответа (SYN-ACK) порт маркируется как уязвимый.
3. Модуль принятия решений (Decision Module) запрашивает действие у оператора.
4. Модуль исполнения транслирует команду в низкоуровневый фильтр пакетов (Netfilter/WFP).

## 2. Нормативно-правовая база и стандарты защиты

Разработка велась с учетом требований регуляторов (ФСТЭК, ФСБ) и международных стандартов ИБ.

### Законодательство РФ:
- **ФЗ-187 (О безопасности КИИ):** Реализация функций инвентаризации и оперативного реагирования на инциденты в критической инфраструктуре.
- **ФЗ-152 (О персональных данных):** Обеспечение технических мер защиты согласно 19 статье (нейтрализация актуальных угроз).
- **Приказ ФСТЭК №17:** Требования к защите ГИС (управление конфигурацией и сетевой фильтрацией).
- **Приказ ФСТЭК №21:** Меры защиты персональных данных в ИСПДн (УПД, АВЗ).

### Нормы защищенности и сертификация:
Программный код следует рекомендациям по безопасной разработке:
- **ГОСТ Р 56939-2016:** Безопасная разработка ПО (общие требования).
- **ISO/IEC 27001:** Механизмы управления доступом к сетевым сервисам.
- **Соответствие уровням доверия (УД):** Структура кода позволяет проводить сертификацию по 4-му уровню доверия ФСТЭК.

---

## 3. Сетевые протоколы и механизмы фильтрации

### Стек используемых протоколов:
- **TCP/IP:** Основной стек для анализа состояния соединений.
- **TCP Connect Scan:** Метод активного зондирования портов через системные сокеты.
- **Netfilter (Linux):** Использование цепочек INPUT/OUTPUT для фильтрации трафика на уровне ядра.
- **WFP (Windows Filtering Platform):** Глубокая инспекция и блокировка трафика через API netsh.

## 4. Структура курсовой работы

1. **Введение:** Актуальность импортозамещения в сфере ИБ и необходимость кроссплатформенных инструментов.
2. **Аналитическая часть:** Сравнительный анализ механизмов защиты Windows и Linux. Обзор законодательства.
3. **Проектная часть:** Описание архитектуры ПО, выбор языка C++ и библиотек WinSock/POSIX.
4. **Реализация:** Алгоритмы сканирования и методы интеграции с системными брандмауэрами.
5. **Тестирование:** Верификация блокировок и анализ производительности.
6. **Заключение:** Перспективы развития и внедрение функций Bluetooth-аудита.

## 5. Инструкция по развертыванию

### Подготовка среды:
- Установка GCC версии 7.0 или выше.
- Наличие прав суперпользователя (sudo) или Администратора.

### Сборка проекта:
**Linux:**
`g++ src/main.cpp src/zeroseg_scanner.cpp -o zeroseg -I include && sudo ./zeroseg`

**Windows:**
`g++ src/main.cpp src/zeroseg_scanner.cpp -o zeroseg.exe -I include -lws2_32`
